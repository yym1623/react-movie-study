### A. 렌더링 / 코드 스플리팅

| **이름**       | **타입** | **설명** |
|---------------|----------|----------|
| `createRoot`  | 함수     | `ReactDOM.createRoot(container).render(<App />)` 형태로 루트 렌더링 시작 |
| `StrictMode`  | 컴포넌트 | 개발 모드에서만 동작하는 검사기. 잠재적인 문제를 두 번 호출 등으로 드러냄 (프로덕션에선 제거) |
| `lazy`        | 함수     | 컴포넌트를 지연 로딩. 사용자가 해당 페이지/컴포넌트에 접근할 때만 동적으로 import |
| `Suspense`    | 컴포넌트 | `lazy` 로드 중이거나 비동기 UI 대기 중일 때 `fallback` UI를 보여줌 (로딩 스켈레톤 등) |


### B. 상태 관리 관련 훅

| **이름**        | **설명** |
|----------------|----------|
| `useState`     | 가장 기본적인 상태 훅. 컴포넌트 로컬 상태를 만들 때 사용 |
| `useReducer`   | 복잡한 상태/여러 액션을 한 곳에서 관리하고 싶을 때 사용 (Redux 축소판 느낌) |
| `useContext`   | 상위에서 만든 Context 값을 하위에서 props 없이 바로 꺼내 쓸 때 사용 |


### C. 사이드 이펙트 / 라이프사이클 훅

| **이름**            | **설명** |
|--------------------|----------|
| `useEffect`        | 렌더링 이후 비동기 호출, 구독, DOM 이벤트 등록 등 “화면에 그려진 뒤” 해야 할 일 처리 |
| `useLayoutEffect`  | DOM이 변경된 직후, 브라우저가 화면 그리기 전에 동기적으로 호출 (레이아웃 측정, 스크롤 제어 등) |


### D. 메모이제이션 / 성능 관련 훅

| **이름**        | **설명** |
|----------------|----------|
| `useMemo`      | 비싼 계산 결과를 기억. 의존성이 안 바뀌면 이전 계산값 재사용 (연산 최적화) |
| `useCallback`  | 함수 자체를 메모이제이션. 자식 컴포넌트에 prop으로 넘길 때 불필요한 리렌더 줄이기 |
| `memo`         | 컴포넌트를 메모이제이션. props가 바뀌지 않으면 다시 렌더링하지 않도록 함 |
| `useTransition`| 상태 업데이트를 “덜 중요한 작업”으로 표시해, UX를 부드럽게 만드는 데 사용 (React 18+) |
| `useDeferredValue` | 어떤 값의 업데이트를 늦춰서, 입력 반응성과 무거운 렌더링을 분리할 때 사용 |


### E. DOM / 참조 관련 훅

| **이름**    | **설명** |
|------------|----------|
| `useRef`   | 렌더 간에 유지되는 “상자” 같은 ref. DOM 요소 접근 또는 mutable 값 저장에 사용 (리렌더 트리거 X) |
| `forwardRef` | 부모가 자식 컴포넌트 내부 DOM에 ref를 직접 전달하고 싶을 때 사용 |


### F. 자주 쓰는 조합 패턴 요약

| **패턴 이름**              | **조합 훅 / 컴포넌트**                     | **예시 / 용도** |
|---------------------------|------------------------------------------|-----------------|
| 기본 상태 + 이펙트        | `useState` + `useEffect`                | 입력값 상태 관리 + 입력이 바뀔 때 API 호출 |
| 리스트 + 성능 최적화      | `memo` + `useCallback` + `useMemo`      | 큰 리스트에서 아이템 컴포넌트를 `memo`로 감싸고, 핸들러는 `useCallback`으로 고정 |
| Context + 전역-ish 상태   | `createContext` + `useContext` + Provider | 테마, 로그인 정보 등 앱 전역에서 필요한 값 전달 |
| 코드 스플리팅             | `lazy` + `Suspense`                     | 라우터에서 페이지 컴포넌트를 lazy 로딩해 첫 로딩 번들 크기 줄이기 |
